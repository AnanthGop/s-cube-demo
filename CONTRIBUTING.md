# Contributing to S-Cube

Thank you for contributing to S-Cube! Please follow these guidelines.

## 🔄 Workflow

### For Collaborators with Write Access:

1. **Create a feature branch** (never commit directly to `main`):

   ```bash
   git checkout -b feature/your-feature-name
   ```

2. **Make your changes** and commit:

   ```bash
   git add .
   git commit -m "Description of changes"
   ```

3. **Push your branch**:

   ```bash
   git push origin feature/your-feature-name
   ```

4. **Create a Pull Request** on GitHub:
   - Go to the repository on GitHub
   - Click "Compare & pull request"
   - Fill out the PR template
   - Request review from @AnanthGop

5. **Wait for approval** - do not merge your own PRs

### For External Contributors:

1. **Fork the repository** on GitHub

2. **Clone your fork**:

   ```bash
   git clone https://github.com/YOUR-USERNAME/s-cube.git
   cd s-cube
   ```

3. **Create a feature branch**:

   ```bash
   git checkout -b feature/your-feature-name
   ```

4. **Make changes and push to your fork**:

   ```bash
   git add .
   git commit -m "Description of changes"
   git push origin feature/your-feature-name
   ```

5. **Create a Pull Request** from your fork to the main repository

## ⚠️ Important Rules

- ❌ **NEVER push directly to `main`**
- ❌ **NEVER force push** (`git push -f`)
- ✅ **Always create a feature branch**
- ✅ **Always create a PR for review**
- ✅ **Wait for approval before merging**
- ✅ **Resolve all review comments**

## 🔍 Code Review Process

All PRs require:

- At least 1 approval from @AnanthGop
- All conversations resolved
- No merge conflicts

## 📝 Commit Message Guidelines

Use clear, descriptive commit messages:

- `Add rent voucher generation feature`
- `Fix cash flow calculation bug`
- `Update expense mapping UI`

## 🆘 Getting Help

If you have questions, please:

- Open an issue on GitHub
- Contact the repository owner
- Check existing documentation

---

**By contributing, you agree to follow these guidelines and respect the code review process.**
